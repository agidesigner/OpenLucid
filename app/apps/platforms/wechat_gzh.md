平台写作规范（微信公众号长文）：

**长度**：1500-3000 字最佳。读者愿意花 5-10 分钟阅读一篇高质量文章。少于 800 字显得轻薄，超过 4000 字阅读完成率骤降。

**内容类型**（选对类型决定节奏）：
- **知识干货**：问题 → 拆解 → 方法论 → 案例 → 总结。信息密度高，小标题多。常带"全流程 / SOP / 思维模型 / 保姆级"字样。
- **情感叙事**：场景 → 冲突 → 转折 → 启示。叙事为主，金句收尾。
- **深度评论**：热点事件 → 独立观点 → 论据 → 行动号召。观点鲜明，反常识，标题常用陈述句（"XX，变了"/"XX 正在吃掉 YY"）。
- **人物故事**：主人公 → 困境 → 选择 → 结果。像小传记，情绪高低起伏。
- **工具合集 / 资源整理**：分类 → 逐项列表 → 场景匹配 → 收藏引导。常带"史上最全 / 全集 / 精选" 字样。

---

**标题公式**（5 种 archetype，选一种执行）：

- **数字型**：`3 个月，我把账号粉丝从 0 做到 10 万` / `36 个运营思维模型` —— 具体数字激活点击
- **疑问型**：`为什么你的小红书总是没流量？` / `谁在买 1000 块的国货洗发水？` —— 陈述句变问句即可，**深度评论类**最爱用
- **工具/方法论型**：`史上最全的运营工具大合集` / `保姆级 SOP：从选题到爆款的 12 步` —— 带"全集 / 保姆级 / SOP / 模型"字样
- **反常识观点型**：`不要再教别人"加班"了，这是新的愚蠢` / `"用户画像"正在毁掉你的产品` —— 挑战共识，但必须在正文兑现
- **陈述+变化型**（虎嗅/36氪高频）：`汽车价格战，突然停了` / `曾经最赚钱的折扣卖场，也开始关店了` —— 主语+逗号+变化动词，冷静但强钩子

**标题禁忌**：标题党 + 内容空洞 = 订阅者批量取关；**感叹号超过 1 个**会被算法打上营销号标签。

---

**开头（前 100 字是生死线）**：立刻告诉读者"看完能得到什么"。删除所有铺垫（"最近我在想...""前几天和朋友聊天..."全是废话）。直接给：
- **场景+痛点**（"凌晨两点，你的订单还卡在审核"）
- **反常识观点**（"'增长'已经是一个过时的 KPI"）
- **数据钩子**（"过去 30 天，87% 的抖音创作者收入在下降"）

**分段节奏**：每段 3-5 行（80-150 字）。**大段文字墙在手机上劝退率极高**。每 2-3 段之后留一空行喘口气。

**小标题节奏**：每 300-500 字一个小标题。让读者扫读也能抓主线。小标题自身必须有吸引力：
- ✅ 我踩过的 3 个坑 / 为什么"用户画像"失效了 / 真正值得抄的那批公司
- ❌ 踩坑总结 / 背景分析 / 总结

**论证密度**：每个观点至少一个**例子或数据**。空洞的"我认为"没有说服力。**具体 > 抽象**，始终。

---

**公众号特有排版约定**：

- **金句卡片 / 划线高亮**：每 500-800 字，把一个核心金句用独立一行 + **加粗**突出，读者容易截图转发。金句 15-25 字最适合。
- **引用块**（`> `）用于：名人/客户原话 / 关键数据出处 / 自己上一篇的观点呼应。
- **表格**比长段落好：3+ 项对比时，一个 Markdown 表格远胜罗列段落。
- **代码块 / 截图位**：用 image_hint 标记。干货型/工具型每 500 字建议一张图（示意图/截图/流程图）。
- **"阅读原文"按钮**：如果有外链资源（下载包/完整文档），结尾一句提示"完整版在阅读原文"。

---

**视觉布局模块**（结构自带排版 —— LLM 自动应用，用户无感知）：

公众号编辑器和绝大多数 markdown 渲染器（mdnice / md2wechat 等）会把下面这些 markdown 结构渲染成视觉块，**输出 markdown 时主动用这些结构能让最终成品自带排版美感**，无需用户手动操作：

- **数字承诺前置块**：开头 hook 之后（首屏 3-5 行内），用**单独一行的粗体**摆一个可扫读的承诺骨架，例如：
  > **3 个月 · 烧掉 20 万 · 5 个踩坑 · 1 个跑通方案**

  作用：手机端首屏停留率显著提升 —— 读者一眼看到"读完能拿到什么"。

- **Timeline 时间线**：当文章包含时间序列（运营复盘 / 项目记录 / 转折历程），把它写成 date-prefixed 列表，**不要塞进段落**：
  ```
  - 2024 年 11 月：发现颈纹，第一次拍照对比
  - 2025 年 1 月：试了 4 款颈霜全部失败
  - 2025 年 3 月：朋友寄来 SDX，开始 28 天记录
  ```
  渲染后是带时间戳的视觉节奏块，比"去年冬天我..."的散文叙事更易读。

- **三栏对比 / Tradeoff 卡**：决策类内容（"该不该用 / 怎么选"）用 markdown 表格做 3 列对比，列名用「**旧做法 / 新做法 / 你应该选**」或「**情况 A / 情况 B / 临界点**」。读者扫一眼即可决策。

- **CTA 引用块**：结尾的"行动建议"和"留白问题"用 `> ` 引用块包起来（**而不是普通段落**）。视觉上独立成块，渲染器会给一条左边粗线 + 缩进底色，扫读时一眼锁定 CTA：
  ```
  > 你准备这周先做哪一条？在评论区告诉我。
  > 如果觉得有用，转发给那个还在为内容产能发愁的合伙人。
  ```

这些都是 markdown 原生语法，公众号编辑器、mdnice、md2wechat 等下游工具都能正确渲染 —— **生成阶段写对结构 = 用户拿到的 markdown 已经自带排版**。

**结尾三件套**（公众号文化）：
1. 核心观点总结（2-3 句） —— 不要摘抄正文，要升华一句
2. 一个**行动建议**或**留白问题**（"你准备这周先做哪一条？"）
3. 互动引导 + 关注引导：**评论区留言** / 转发朋友圈 / 点"在看"。不要强制关注。

**风格**：
- **书面语为主但不学术化**，像跟一位认真读文章的朋友讲话
- **禁用虚指"我们/大家"**；用"你"或明确身份（"如果你在做 SaaS..."）建立对话感
- **金句是公众号的灵魂** —— 每 500 字要能截一句发朋友圈
- **数据要带出处**：不能写"据统计"就塞一个数，要写"据 QuestMobile 2025Q3 报告..."（可以脚注 / 括号里标）—— 否则可信度归零

**SEO 关键词**：标题和首段自然嵌入 1-2 个核心关键词，帮助微信"搜一搜"曝光。**但不要堆砌** —— 搜一搜的语义理解远强于关键词密度匹配。

**封面图 + 摘要**：订阅号展示页（信息流）**只显示封面 + 标题 + 摘要 4 行**。用户决定是否点开就看这一屏。封面图要：有辨识度 / 避免大段文字堆 / 与标题呼应但不重复。摘要用 30-40 字补充标题的"承诺"。
