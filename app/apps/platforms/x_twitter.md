Platform writing rules (X / Twitter Thread):

**When a thread works**: frameworks, data stories, personal transformations, contrarian takes, step-by-step guides, case studies. **When it doesn't**: generic tips, broad opinions, product promos (those belong in single tweets).

**Format**: **10-25 tweets for most threads; top performers often run 20-30+**. Median of top threads ≈ 17 tweets. Each tweet 200-280 chars unless Premium (then up to ~500 chars for dense analysis tweets). Each tweet must **stand alone when retweeted** while connecting to the next.

**Tweet 1 (the hook)** — the retweet-magnet. 78% of top threads end the hook with an explicit payoff phrase (`"Here's how:"`, `"Here's why:"`, `"Here's the breakdown:"`, or a colon-cliff). Optionally append `🧵` or `👇` (~47% of tops do). Pick one archetype:

- **Bold claim + payoff pivot**: "Most founders fail at pricing. Here's why:"
- **Surprising data + promise**: "$1.2M ARR, 0 employees. Here's the stack:"
- **Counterintuitive**: "Growing slower made our revenue 3x. Here's what we did:"
- **Named framework**: "The Poison Arrow Principle. Why most advice on overthinking fails, and what works instead:"  (Sahil Bloom-style — crediting a named model in the hook is a top-performer archetype.)
- **Case-study / rags-to-riches lead**: "$100B retailer. Sells to the whole world except China. Here's how Shein did it:"  (Lead with surprising outcome/stat, promise the backstory.)
- **Avoid**: naked "Here are 5 things about X" — but "5 principles that will 10x your Y. Here's #1:" works because it promises a specific payoff.

**Tweet 2-3 (setup)**: Briefly explain WHY this matters or WHO should care. Build the payoff promise.

**Middle tweets (the meat)**:
- One idea per tweet — don't cram.
- **Numbering is conditional**, not universal (~56% of top threads use it):
  - Use numbered markers (`1/`, `#1`, `Step 1`) for **list / framework / case-study** threads.
  - **Skip numbering** for **storytelling / rebuttal / news** threads — it breaks narrative flow.
- Short sentences. Break long thoughts across tweets.
- Use "Here's why:" / "The trick:" / "What most miss:" as mid-thread continuation hooks.
- Line breaks within a tweet for rhythm (blank lines).

**Images / charts / screenshots**: For visual, data, or case-study threads, include media **every 2-3 tweets** (~1 image per 3 tweets is the top-performer norm; 72% of tops have ≥1 image). Pure-text threads work for story/opinion, but list threads without visuals consistently underperform.

**Premium long-form tweets** (if user has X Premium): combine related points into fewer, denser tweets — e.g., 8 takeaways in 2 Premium tweets, bolding the lead line of each takeaway. **Don't mix Premium length with `1/ 2/` list numbering** — pick one format.

**Emoji use**: Sparingly. ✅ / ❌ / → are useful as visual markers. Decorative emojis (🔥💯) look amateur unless used once as a thread-marker in the hook.

**No hashtags mid-thread** — they break reading flow. Save 1-3 for the last tweet if at all.

**Closing 2-3 tweets** (not one):
1. **TL;DR / recap** — a numbered or bulleted summary of every point covered.
2. **Primary CTA** — strongest pattern for top performers is a **newsletter / lead-magnet / course link** (~56% of top threads: "Full playbook here: [link]"). Secondary: "Retweet the first tweet if you learned something" (~19%) or "Follow me for more [topic]" (~16%).
3. **Optional**: a soft follow prompt or a second RT-ask.

Rebuttal / news / opinion threads can end without any CTA (~22% of tops) — let the closing beat land.

**Style**: Conversational, confident, punchy. Treat each tweet like ad copy, not an essay paragraph. Remove every filler word. Read each tweet aloud — if it sounds like a blog post, rewrite.

**Output format**: In sections, each section's `text` is ONE tweet. Use `\n\n` within a tweet for line breaks. Do NOT put multiple tweets in one section.
