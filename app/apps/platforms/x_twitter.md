Platform writing rules (X / Twitter Thread):

**When a thread works**: frameworks, data stories, personal transformations, contrarian takes, step-by-step guides. **When it doesn't**: generic tips, broad opinions, product promos (those belong in single tweets).

**Format**: 5-15 tweets, each 200-280 characters. Each tweet must **stand alone when retweeted** while connecting to the next.

**Tweet 1 (the hook)**: This is the retweet-magnet. Use one of:
- **Bold claim**: "Most founders fail at pricing. Here's why."
- **Surprising data**: "$1.2M ARR, 0 employees. I'll show you the stack."
- **Counterintuitive**: "Growing slower made our revenue 3x."
- **Avoid**: "Here are 5 things about..." (too generic, no hook)

**Tweet 2-3 (setup)**: Briefly explain WHY this matters or WHO should care. Build the payoff promise. Keep it short.

**Middle tweets (the meat)**:
- One idea per tweet — don't cram
- Use numbered markers (1/, 2/, 3/) for readability
- Short sentences. Break long thoughts across tweets.
- Use "Here's why:" / "The trick:" / "What most miss:" as continuation hooks
- Line breaks within a tweet for rhythm (use blank lines)

**Emoji use**: Sparingly. ✅ / ❌ / → are useful as visual markers. Decorative emojis (🔥💯) look amateur.

**No hashtags mid-thread** — they break reading flow. Save 1-3 for the last tweet.

**Closing tweet**: One-sentence summary + explicit CTA (follow for more, reply with your experience, retweet the first tweet).

**Style**: Conversational, confident, punchy. Treat it like copy, not an essay. Remove every filler word. Read each tweet aloud — if it sounds like a blog, rewrite.

**Output format**: In sections, each section's `text` is ONE tweet. Use `\n\n` within a tweet for line breaks. Do NOT put multiple tweets in one section.
