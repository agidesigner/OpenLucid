Platform writing rules (SEO-optimized article for blog / website):

**Purpose**: Long-form article designed to rank in Google/Bing/Baidu for a specific search intent. Published on the user's own blog or website. Success = showing up on page 1 for the target keyword + driving organic traffic.

**Target length**: 1500-3000 words. Under 1000 words rarely ranks for competitive keywords. Over 4000 words dilutes relevance.

**Search intent match**: Before writing, identify the user's likely search intent:
- **Informational**: "how to / what is / why does" → educational content, explain concepts
- **Commercial investigation**: "best X for Y / X vs Y / X review" → comparison, pros/cons, recommendations
- **Transactional**: "buy X / X pricing / X discount" → conversion-focused, clear offer
- **Navigational**: "brand name + feature" → straightforward, product-focused

**Title (H1)**:
- Include the target keyword in the first 60 characters
- Number / year / specific benefit increases CTR: "7 SEO Tactics for 2026 (That Actually Work)"
- Match the search intent (don't promise "best X" and deliver a tutorial)

**Meta description** (include in section metadata if possible):
- 150-160 characters
- Include the keyword once, naturally
- Write a compelling summary that answers "why should I click?"

**Structure**:
- **Intro** (first 100-150 words): Hook + problem statement + what the reader will learn. Include the target keyword naturally in the first 100 words.
- **Body**: Break into H2 / H3 sections with semantic keywords in headings. Each H2 should answer a specific sub-question.
- **Short paragraphs**: 2-4 sentences per paragraph. SEO algorithms prefer readability.
- **Lists and tables**: Break up walls of text. Google favors these for featured snippets.
- **FAQ section** at the end: Answer 3-5 related questions. Directly addresses "People Also Ask" and wins snippet real estate.
- **Conclusion**: Recap key points + clear next action (but not aggressive CTA — SEO content converts via trust, not pushiness).

**Keyword usage**:
- Target keyword: 5-10 times in a 2000-word article (1-2% density max)
- Semantic variations: use naturally throughout (e.g., for "content marketing": "content strategy", "marketing content", "content creation")
- **Never** keyword stuff — Google penalizes it
- Place keyword in: title, first paragraph, at least one H2, URL slug, image alt text

**Readability**:
- Grade-level target: 8-10 (accessible, not dumbed down)
- Active voice > passive voice
- Concrete examples > abstract theory
- First person ("we", "I") or second person ("you") increases engagement

**E-E-A-T signals** (Google ranking factors):
- **Experience**: Share first-hand experience ("When I ran this test on 200 sites...")
- **Expertise**: Show technical/domain knowledge through specific details
- **Authoritativeness**: Cite data, link to authoritative sources
- **Trustworthiness**: Admit limitations, cite sources, include author context

**Internal linking opportunities**: When writing, note 2-3 places where a related article on the same site would fit (mark them in the output so the user can add links later).

**Image suggestions**: Describe 2-4 ideal images (in image_hint fields) — hero image at top, illustration for complex concepts, screenshot/chart for data, and one visual for the CTA area. Include alt text suggestions.

**Platform-agnostic output**: This content works on WordPress, Medium, Ghost, Substack, WeChat 公众号, 知乎, and any blog CMS. Just paste and format.
